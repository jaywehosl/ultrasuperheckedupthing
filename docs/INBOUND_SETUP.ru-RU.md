# Создание инбаундов на turnkey-установке (реверс-прокси + cookie-gate)

> 🇬🇧 English version: [INBOUND_SETUP.md](INBOUND_SETUP.md)

Гайд именно под **рекомендуемое turnkey-развёртывание** (см.
[README](../README.ru-RU.md) и [шаблон nginx](nginx-template.ru-RU.md)). Описаны
два инбаунда, которые оператор создаёт **в панели**:

1. **VLESS · TCP · Reality** с доменом **selfsteal/selfsni**
2. **Hysteria2**

Общее управление инбаундами/клиентами — в доке upstream
[3X-UI](https://github.com/MHSanaei/3x-ui); здесь только то, что специфично для
нашей установки.

---

## Почему здесь обязателен External Proxy + `forceTls = same`

На этом развёртывании публичный TLS-фронт на **:443** — это **Xray VLESS
Reality**, который маскируется под *selfsteal*-домен и реле не-VLESS трафик в
nginx (панель / подписка / заглушка) через unix-сокет. Из-за этого фронта адрес,
на который инбаунд *биндится*, **не равен** адресу, куда должны подключаться
клиенты, и панель не всегда может сама вывести публичную точку.

Поэтому для этих инбаундов вы добавляете строку **External Proxy**, которая
переопределяет **dest + port (+ SNI)** в каждой генерируемой шэр-ссылке /
подписке, а `forceTls = same` сохраняет собственную security инбаунда (Reality /
TLS) в ссылке неизменной. Кратко:

> **External Proxy → `Force TLS = same`, `Address = <публичный домен>`,
> `Port = 443`.** Именно это заставляет ссылки указывать клиентам на реальную
> публичную точку с правильным SNI.

Поля (из строки External Proxy в панели): `Force TLS` (`same` / `tls` / `none`),
`Address` (dest), `Port`, `SNI`, `Remark`.

## 1. VLESS · TCP · Reality + selfsteal (selfsni)

**Inbounds → Add inbound.**

1. **Протокол:** VLESS.
2. **Listen IP:** пусто / `0.0.0.0` (или `127.0.0.1`, если Xray фронтит через
   сокет — по вашей установке). **Порт:** `443`.
3. **Transmission:** TCP.
4. **Security:** Reality.
5. **Настройки Reality:**
   - **Dest / Target:** selfsteal-апстрим вашей установки — unix-сокет
     (`/dev/shm/xui.sock`) или фолбэк `127.0.0.1:8443`, с **xver = 1** (PROXY
     protocol), чтобы nginx за ним получал реальный IP клиента.
   - **Server Names (SNI):** ваш **selfsteal-домен** (хост-заглушка, под который
     маскируется Reality — тот же `SELFSTEAL_DOMAIN` из установки).
   - **Ключи:** сгенерируйте пару ключей Reality (в форме есть кнопка генерации);
     добавьте один или несколько **shortId**.
   - **uTLS / fingerprint:** `chrome` (разумный дефолт).
6. **Добавьте клиента** (UUID сгенерируется; задайте email/flow по нужде —
   для VLESS Reality TCP типичен `xtls-rprx-vision`).
7. **External Proxy (обязательно):** добавьте одну строку:
   - **Force TLS:** `same`
   - **Address:** ваш **selfsteal-домен**
   - **Port:** `443`
   - **SNI:** selfsteal-домен (как в Server Names)
   - **Remark:** например `reality`
8. **Save.**

Теперь шэр-ссылка / подписка отдаёт
`vless://…@<selfsteal-домен>:443?security=reality&sni=<selfsteal-домен>&fp=chrome&pbk=…&sid=…&flow=xtls-rprx-vision`
— клиенты подключаются к публичному Reality-фронту на 443, а неавторизованные
пробы видят только сайт-заглушку.

## 2. Hysteria2

Hysteria2 — это **UDP/QUIC** со **своим TLS** — он **не** идёт через TLS-фронт
nginx, поэтому ему нужен отдельный UDP-порт и сертификат.

**Inbounds → Add inbound.**

1. **Протокол:** Hysteria2.
2. **Listen IP:** `0.0.0.0`. **Порт:** свободный **UDP**-порт (например, `443/udp`
   может сосуществовать с `443/tcp` Reality-фронтом, или выберите другой, напр.
   `8443`).
3. **TLS:** укажите **cert / key** на сертификаты, выпущенные установщиком
   (Let's Encrypt для вашего домена — путь, который напечатала установка). **SNI /
   домен** — домен этого сертификата.
4. **(Опц.) obfs:** включите обфускацию Salamander с паролем для доп. маскировки.
5. **Пропускная способность:** задайте up/down, если нужны подсказки клиенту.
6. **Добавьте клиента** (пароль сгенерируется).
7. **External Proxy (тоже обязательно):** добавьте строку, чтобы ссылка указывала
   на публичный хост:
   - **Force TLS:** `same`
   - **Address:** домен, чей сертификат использовали
   - **Port:** UDP-порт из шага 2
   - **SNI:** этот домен
   - **Remark:** например `hy2`
8. **Откройте UDP-порт** на фаерволе / в security group провайдера.
9. **Save.**

Шэр-ссылка отдаёт
`hysteria2://<пароль>@<домен>:<udp-порт>?sni=<домен>[&obfs=salamander&obfs-password=…]`.

## Проверка

- **Copy URL / QR** в строке клиента или откройте ссылку **подписки** — убедитесь,
  что хост — ваш **публичный домен** (а не `127.0.0.1` / внутренний бинд), а
  SNI/порт — те, что задали. Если видите внутренний адрес — строка **External
  Proxy** отсутствует или неверна.
- Браузер на **selfsteal-домене** должен показать заглушку; панель остаётся
  невидимой без cookie-gate ссылки.

> Напоминание: не меняйте после развёртывания **порт, путь или домен**
> панели/подписки без нужды — встроенная защита предупредит, и это может сломать
> контракт прокси. См. [Руководство пользователя](USER_GUIDE.ru-RU.md).
