# Community Panel — Руководство пользователя

Здесь описаны **только фичи, которые Community Panel добавляет поверх стокового
[3X-UI](https://github.com/MHSanaei/3x-ui)**. Инбаунды, клиенты, роутинг, движок
подписок, Telegram-бот и т.п. — без изменений, смотрите upstream-доку. Новое
здесь — **интерфейс**, **система тем**, **система уведомлений** и несколько
фич **безопасности/удобства**.

> Локализация **временно только английская** — многие модалки ещё не переведены,
> поэтому English стоит по умолчанию и единственным пунктом во всех дропдаунах.

> 🇬🇧 English version: [USER_GUIDE.md](USER_GUIDE.md)

## 1. Appearance (темизация) — `Settings → Appearance`

Две вкладки: **Styles** и **Notifications**.

### Styles
Контролы с живым превью; изменения применяются мгновенно и сохраняются **на
сервер** (поэтому видны на странице логина, странице подписки и для всех админов):

- **Theme** — База Light / Dark / Ultra-dark.
- **Glass & blur** — размытие и насыщенность стеклянных поверхностей.
- **Shape & depth** — масштаб скругления углов и интенсивность теней.
- **Background Wallpaper** — Aura (амбиентный градиент), сплошной цвет или
  загруженное изображение (с затемнением и блюром).
- **Palette** — Primary (управляет акцентами, амбиентной аурой, тоглами,
  выделениями) + Success / Warning / Error. Есть проверка контраста.
- **Effects** — интерактивное поле частиц: вкл/выкл, плотность, скорость, режим
  цвета (Multicolor / Primary / Monochrome), тумблер hover-glow.
- **System Fonts** — Sans / Display / Mono или загрузка своего шрифта.

**Действия справа:** **Save** (на сервер), **Export** / **Import** (JSON-файл
темы), **Reset** (к заводским дефолтам Community Panel).

**Пресеты:** *Community Panel* (заводской) и *Nordic Frosted*.

**Подсказки**
- Смена **Primary** перекрашивает весь UI вживую — включая частицы и амбиентное
  свечение — без перезагрузки.
- Красный тост Save («applied to this browser only») = сервер не принял (локально
  всё равно применится). На логине тема всегда локальная — уедет на сервер после
  входа.

### Notifications (вкладка)
См. §2.

## 2. Уведомления

Уведомления появляются в двух синхронных местах:
- **строка статус-бара** (открывается **колокольчиком** в шапке) и
- **счётчик-бейдж** на колокольчике.

Всё также пишется в **журнал истории** (Appearance → Notifications → History).
Всплывающие тосты тоже зеркалятся в историю — ничего не теряется.

### Источники алертов (живые состояния)
Тумблеры в **Appearance → Notifications → Alert sources**:
- **Security warnings** — дефолтный порт / base path / доступ по HTTP и т.п.
- **Xray-core health** — ядро упало / остановлено.
- **Restart reminders** — нужен рестарт панели / ядра после сохранения.

У каждого в строке есть **✕**. Гашение живого алерта **заглушает именно его
навсегда** и отправляет в историю, где кнопкой **Restore** его можно вернуть. (Мы
за реверс-прокси + cookie-gate, поэтому предупреждения о «дефолтном порту/пути»
обычно просто шум, который можно убрать.) **Clear** в истории сохраняет записи,
по которым ещё доступен Restore.

### Пороговые сенсоры (живые, реального времени)
**Appearance → Notifications → Threshold sensors.** Каждый = тумблер + порог.
Читают живой статус сервера (~2 с) и показывают строку **с текущим значением**,
пока выше порога; строка **обновляется вживую** и **исчезает сама**, когда
значение падает. Гашение прячет на **текущий эпизод** — перевзводится
автоматически, когда значение упадёт, так что следующий всплеск снова уведомит.
(Это состояния, а не события, поэтому в историю не спамятся.)

Сенсоры: **CPU %**, **Memory %**, **Disk %**, **TCP-сокеты**, **UDP-сокеты**,
**Uptime reminder** (дни), **Client offline** (клиент, который *был* онлайн и
пропал на N часов).

Дефолты Community Panel: CPU 55 · RAM 60 · Disk 30 · TCP 1000 · UDP 1000 ·
Uptime 7 д · Client-offline 12 ч — все включены.

### Security log
Целевой наблюдатель за логом (по умолчанию выкл). Он **не** дампит весь лог —
показывает **неудачные входы в панель** (`username` / IP / причина), схлопывая
брутфорс-серию в одно уведомление. (Дальше — IP-лимит, SSH.)

## 3. Защита от самолок-аута

Изменение настройки, влияющей на **то, как достигается панель/подписка** —
**порт, адрес прослушивания, домен, base path, TLS-сертификат** панели/подписки —
открывает модалку жёсткого подтверждения перед сохранением, потому что за
реверс-прокси + cookie-gate неверное значение может **навсегда закрыть доступ** без
отката из панели. Модалка:
- перечисляет, какие именно критичные поля изменены,
- держит **таймер 15 секунд**,
- требует **галку подтверждения**, и только потом активирует кнопку, и
- предлагает в один клик **«Сделать бэкап»** (экспорт БД).

---

## 4. Бэкап и восстановление — `дашборд → Backup`

- **Export database** / **Export migration** — скачать снапшот.
- **Restore (импорт БД)** — загрузить `.db` (SQLite) / `.dump` (Postgres).
  Восстановление подменяет БД и рестартит ядро *внутри запроса*, поэтому панель
  на миг пропадает. Community Panel обрабатывает это аккуратно: ждёт возвращения
  панели, затем сообщает об **успехе уведомлением** и перезагружается. (Раньше это
  выглядело как вечный спиннер с ошибкой сети.)

Делайте бэкап перед любым рискованным изменением — модалка (§3) даёт кнопку прямо
там.

---

## 5. Информация о клиенте

Инфо-модалка клиента переработана для компактности: полоса статуса (онлайн /
включён / email + last-online), копируемые строки кредов (subId / UUID / пароль),
двухколоночная сетка метрик (трафик, истечение, IP-лимит, IP-лог,
создан / обновлён), привязанные инбаунды — плюс ссылки подписки и QR-коды.

---

## Заметка о рекомендуемом развёртывании

Community Panel заточена под установку **turnkey реверс-прокси + cookie-gate**
(см. [README](../README.ru-RU.md)). После такого развёртывания не меняйте
**порт, путь или домен** панели/подписки без нужды — защита (§3) предупредит, но
безопаснее: сделать бэкап, менять в окно обслуживания и держать под рукой
секретную cookie-gate ссылку.
