# Community Panel — Архитектура и дизайн

Структурный гайд: **как** собрана Community Panel (тех. часть) и **почему** (идеи).
Предполагается знакомство с upstream [3X-UI](https://github.com/MHSanaei/3x-ui);
здесь описано только то, что мы изменили или добавили.

> 🇬🇧 English version: [ARCHITECTURE.md](ARCHITECTURE.md)

---

## Часть I — Теория (идеи)

**Учебный проект, собранный с ИИ.** Цель была не «панель получше», а взять
реальную нетривиальную кодовую базу и **перестроить весь UI** — как практическое
исследование ИИ-ассистированной разработки. Go-бэкенд намеренно держится близко к
upstream, чтобы панель работала; расхождение — фронтенд и обвязка сборки/UX.

**Единый визуальный язык.** Один глассморфный глянцевый стиль (матовые
`backdrop-filter`-поверхности, амбиентная аура, WebGL-частицы), применённый
консистентно — вместо стокового Ant Design.

**Всё настраивается, ничего не захардкожено.** Визуальные значения идут из
дизайн-токенов, поэтому одна страница Appearance перенастраивает весь UI (цвета,
стекло, радиусы, тени, шрифты, фон, эффекты). Когда что-то «не реагировало»,
правильным фиксом всегда было провести значение через токен, а не частный случай.

**Законченные инструменты, а не фичи ради фич.** Установщик/CLI смоделированы по
eGames [remnawave-reverse-proxy](https://github.com/eGamesAPI/remnawave-reverse-proxy)
как пример *цельного, реально полезного* инструмента. Мы переиспользовали его
скрипты/CLI как образец — не его идеи.

**Безопасность через «отсутствие».** Рекомендуемое развёртывание прячет панель за
реверс-прокси + cookie-gate, делая её невидимой для сканеров (см. Часть II).

## Часть II — Техническая

### Фронтенд
- React 19 + TypeScript + Vite, отдаётся Go-бинарником через `//go:embed`.
- Ant Design постепенно заменён самописной библиотекой **DS** (диалоги, таблицы,
  формы, вкладки, меню, карточки, тосты) на примитивах **Radix UI** +
  **TanStack Query/Table**, CodeMirror для JSON-редактора, Zod для валидации.
- REST API обёрнут генерируемым OpenAPI-клиентом.

### Движок тем
- `frontend/src/styles/tokens.css` — единый источник истины (дизайн-токены):
  цвета, размытие/насыщенность стекла, рампа радиусов, тени, анимация, амбиентные
  `--aura-*` (доминирующая доля следует за `--color-primary`) и производные
  акценты (`--color-primary-strong` / `-soft` через `color-mix` / rgb).
- `frontend/src/theme/themeApply.ts` — `themeToCss()` (чистая) строит оверрайд
  `:root{…}` в один управляемый `<style id="uup-theme-overrides">`, и весь UI
  перерисовывается мгновенно. Разворачивает `--radius-scale` по рампе, масштабирует
  альфы теней через `--shadow-intensity` (`calc()`), эмитит `@font-face` для
  загруженных шрифтов и экспортирует `DEFAULT_THEME` (заводская тема).
- **Server-canonical хранение.** Тема — Go-настройка `panelTheme`, отдаётся
  публично на `/theme.json`, сохраняется через `POST /panel/setting/theme` (JSON —
  иначе axios form-кодирует). Без вспышки на старте: Go инлайнит `window.X_UI_THEME`
  в HTML до запуска JS; страница подписки зеркалит тему так же. `bootstrapTheme()`
  предпочитает инжект, затем кэш localStorage, затем `DEFAULT_THEME` на свежей
  установке.

### Подсистема уведомлений
- `frontend/src/stores/notificationStore.ts` — стор (`useSyncExternalStore`):
  `history`, `active` (события), `dismissed` (заглушённые алерты), `sensorAcked`,
  префы (`prefs`, `sensors`, `logWatch`) — всё в localStorage.
- **Две модели.**
  - *Живые состояния* (security/xray/restart и статус-сенсоры
    CPU/RAM/диск/TCP/UDP/uptime) **вычисляются** в `useNotifications` из статуса
    через `lib/notifications/statusSensors.ts` — показывают текущее значение,
    обновляются (~2 с), исчезают сами. Гашение = ack-до-снятия; в историю не
    пишутся (без спама).
  - *События* (failed-login из лог-вотчера, client-offline) используют `active` +
    `pushEvent(dedupKey)` и **edge-triggering** (срабатывание на переходе
    false→true, перевзвод при сбросе), поэтому повтор = новое уведомление, а
    гашение не глушит правило навсегда.
- Headless-вотчеры (`SensorWatcher`, `ClientOfflineWatcher`, `LogWatcher`)
  монтируются один раз в `PanelLayout`. DS-тост-стор зеркалит строковые тосты в
  историю. Строка статус-бара + колокольчик показывают живые состояния + события.

### Безопасность и устойчивость
- **Защита от самолок-аута** (`DangerConfirmModal`) гейтит Save на критичных полях
  (порт/listen/домен/base-path/cert панели/подписки) — таймер + галка + бэкап.
- **Восстановление** (`BackupModal`) трактует импорт как разрушительный рестарт:
  опрашивает `/server/status`, пока панель не вернётся, затем сообщает об успехе —
  вместо зависания на запросе, который прокси рвёт во время восстановления.
- Глобальный axios-интерсептор на 401 перезагружает на логин; сохранение темы
  передаёт `skipAuthRedirect`, чтобы 401 на (неавторизованном) логине не
  перезагружал страницу.

### Установка (turnkey реверс-прокси + cookie-gate)
`install.sh`, режим A: запрашивает три разных домена (панель, подписка,
selfsteal/Reality-заглушка) → выпускает сертификаты Let's Encrypt (HTTP-01) →
пишет конфиг Nginx с **3 SNI-вхостами на одном порту 443** → ставит панель за
**cookie-gate**, который без секретного cookie отдаёт заглушку → рестарт. Итог:
один порт, без порта/пути в URL, и панель, не отвечающая сканерам. Поэтому и
существует защита от самолок-аута — смена порта/пути/домена ломает контракт прокси.

## Где что лежит

| Область | Путь |
|---|---|
| Дизайн-токены | `frontend/src/styles/tokens.css` |
| Движок тем | `frontend/src/theme/` (`themeApply.ts`, `themeStorage.ts`) |
| Страница Appearance | `frontend/src/pages/appearance/` |
| Стор уведомлений | `frontend/src/stores/notificationStore.ts` |
| Сенсоры / вотчеры | `frontend/src/pages/index/` (`SensorWatcher`, `ClientOfflineWatcher`, `LogWatcher`), `frontend/src/lib/notifications/` |
| Строка статус-бара | `frontend/src/pages/index/NotificationsBar.tsx` |
| Модалка безопасности | `frontend/src/components/ui/DangerConfirmModal.tsx` |
| Бэкап / восстановление | `frontend/src/pages/index/BackupModal.tsx` |
| Установщик / CLI | `install.sh`, `x-ui.sh`, `docs/nginx-template.md` |
