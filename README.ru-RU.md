# Community Panel

Community-форк [3X-UI](https://github.com/MHSanaei/3x-ui) — веб-панели управления
[Xray-core](https://github.com/XTLS/Xray-core) — с **полностью переписанным
фронтендом на React**. Go-бэкенд держится близко к upstream; переработан именно
интерфейс.

## Что это и зачем

У 3X-UI зрелый бэкенд, но фронтенд на Ant Design. Цель форка — **не** «сделать
панель лучше», а **научиться**, перестроив весь UI реального проекта:

- увод фронтенда **с Ant Design** на небольшую самописную библиотеку компонентов;
- единый цельный визуальный язык (глассморфизм, глянец);
- всё это **в плотном сотрудничестве с ИИ** — как практическое исследование
  ИИ-ассистированной разработки.

Бэкенд (Go + Xray-core, REST API, движок подписок, БД) намеренно держится близко
к upstream 3X-UI; основные расхождения — фронтенд и обвязка сборки/UX.

### На чём собрано

- **Бэкенд:** Go, Xray-core, серверная часть/API от upstream 3X-UI.
- **Фронтенд:** React 19, TypeScript, Vite, Radix UI, TanStack Query & Table,
  CodeMirror (JSON-редактор), i18next (пока только English — остальные локали
  временно отключены, пока переводятся модалки), Zod.
- **Тулинг:** Vitest + Testing Library, генерируемый OpenAPI-клиент, релизный
  пайплайн на GitHub Actions.

## Быстрый старт

На свежем Linux (amd64 / arm64) VPS запустите установщик:

```bash
bash <(curl -Ls https://raw.githubusercontent.com/jaywehosl/community_panel/main/install.sh)
```

Он скачает свежий бинарник из **GitHub Releases**, поставит systemd-сервис `x-ui`
и зависимости, и предложит режимы развёртывания ниже. (Канал prerelease — самая
свежая сборка, включая предрелизы: замените `install.sh` → `install-prerelease.sh`.)

### ✅ Рекомендуется: turnkey реверс-прокси + cookie-gate

Когда установщик спросит, выберите **«Реверс-прокси / Nginx с доступом по
доменам»** и стиль доступа **«Cookie-gate (секретная ссылка с cookie-авторизацией)»**.
Вы указываете три **разных** домена — панель, подписка и *сайт-заглушка
(selfsteal / Reality)* — а установщик делает всё под ключ:

- выпускает сертификаты Let's Encrypt (HTTP-01),
- настраивает **Nginx с 3 SNI-вхостами на одном порту 443** (без портов в URL),
- ставит панель за **cookie-gate** и печатает **секретную cookie-gate ссылку —
  сохраните её** (ею панель открывается в браузере).

**Почему это рекомендуемый способ:**

| Без него | С реверс-прокси + cookie-gate |
|---|---|
| Панель на кастомном порту/пути, доступна любому, кто её найдёт | Единый **443**, порт/путь не светятся; чистые доменные URL |
| Дефолтный порт/путь фингерпринтятся → сканеры и брутфорс находят логин | Без секретного cookie панель **не отвечает** — сканер видит **заглушку**, а не логин. Панель фактически невидима |
| TLS на вас | TLS терминирует Nginx, авто-выпуск и продление |
| Логин/cookie могут идти по plain HTTP | Всегда HTTPS |
| Прокси — отдельная ручная морока | Одна установка поднимает панель + подписку + Reality-заглушку вместе |

Именно под это панель и заточена — например, встроенная защита **«это изменение
может закрыть вам доступ»** существует потому, что смена порта/пути/домена ломает
контракт прокси. Подробности — в [Руководстве пользователя](docs/USER_GUIDE.ru-RU.md)
и [шаблоне nginx](docs/nginx-template.md).

> ⚠️ **Сохраните секретную cookie-gate ссылку**, которую печатает установщик. Без
> неё вы не попадёте в панель.

### Чистый бинарник (продвинуто — за своим прокси / SSH-туннелем)

Если у вас свой реверс-прокси или SSH-туннель, выберите чистую установку и
привяжите панель к `127.0.0.1`. Установщик предупредит, если панель окажется
доступна по plain HTTP.

### Локальный запуск (режим разработчика)

Требуется **Go 1.26+** и **Node.js 18+**.

```bash
git clone https://github.com/jaywehosl/community_panel.git
cd community_panel
chmod +x run.sh
./run.sh
```

Только фронтенд с hot reload:

```bash
cd frontend
npm install
npm run dev
```

### Сборка из исходников

```bash
cd frontend && npm install && npm run build && cd ..
go build -o x-ui main.go
sudo chmod +x deploy.sh && sudo ./deploy.sh
```
